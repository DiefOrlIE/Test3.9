Paper Reading Card

Learning to Reason
under Off-Policy Guidance

这篇论文的核心命题很直接：如果一个模型只能从自己刚生成出来的推理轨迹中继续强化，它通常只能把“本来会一点点的能力”放大，而很难学到真正超出自己原始能力边界的新推理模式。作者提出的 LUFFY 就是在回答这个问题。

主题：RLVR 推理训练 关键词：Off-policy Guidance 主方法：Mixed-Policy GRPO 关键补丁：Policy Shaping

论文版本：arXiv v5，2025-06-22。默认主实验基于 Qwen2.5-Math-7B，重点验证数学推理训练，也考察了分布外泛化和弱模型训练场景。

一句话结论

LUFFY 通过把强模型的推理轨迹混入 RLVR，让模型在“模仿强者”和“保持自身探索”之间形成动态平衡。

为什么值得读

它不仅追求分数提升，更在讨论一个深层问题：强化学习能不能帮助模型跨过自身初始能力上限。

你会在文中反复见到

RLVR：Reinforcement Learning with Verifiable Rewards，意思是“带可验证奖励的强化学习”，奖励通常只看最终答案对不对。

On-policy：训练只用当前模型自己采样出来的数据。

Off-policy：训练时允许使用别的策略或别的模型产生的数据。

这份网页怎么读

不是逐句翻译论文，而是保留论文主线和核心公式，再为每个第一次出现的术语补解释，并告诉你“这节真正想说什么”。

最重要的预期

如果你现在几乎没系统学过强化学习，也没关系。先抓住“为什么作者觉得纯 on-policy 不够”，再读公式，理解会顺很多。

Stage 01

导读总览

先建立全局地图，避免一上来就淹没在术语里。

这篇论文最值得你先记住的，不是某个公式，而是一个“训练哲学”的变化。过去很多 大推理模型 的训练思路，是让模型自己反复做题、得到奖励、继续强化；作者认为这种方式有天然上限，因为模型只能在“自己已经摸得到的区域”里打转。

LUFFY 的出发点是：当模型自己不会做时，不应该一味等它自己试出来，而应该允许它看见更强模型的成功推理轨迹。这样，训练不再只是“自己强化自己”，而是带着外部支架去学。

纯 On-policy RL
→
只能强化自己会一点的东西
→
容易撞到能力上限
LUFFY
→
引入强模型轨迹
→
模仿与探索并行
这篇论文真正的研究问题

不是“怎么再涨几分”，而是“RLVR 能否帮助模型学到超出初始能力边界的推理模式”。这是你后面读所有实验时都要记住的主线。

你可以先带着这三个问题读
  1. 纯 on-policy 为什么会卡住？
  2. off-policy 数据怎么混进 RL，而不是只做 SFT？
  3. 为什么模仿强模型不会直接把探索能力弄没？
Stage 02

读前背景补课

把后面会频繁出现的概念先搭起来。

什么是 LRM

LRM 是 Large Reasoning Model，意为“大推理模型”。它强调的不是普通对话，而是多步推理、反思、自我修正等能力。

什么是 CoT

CoT 是 Chain-of-Thought，直译“思维链”。它指模型把中间推理步骤显式写出来，而不是只给最终答案。

什么是 Verifiable Reward

可验证奖励 指奖励不依赖人工主观打分，而是能程序化判断对错，比如数学题最终答案是否正确。

你需要的最小强化学习直觉

强化学习 可以先粗糙理解成：模型先尝试产生一个动作序列，再根据奖励信号调整未来更倾向于哪些动作。对于推理模型来说，一个动作通常可以理解为“在某一位置生成哪个 token”。

Token 是模型生成文本时的基本离散单元。你可以先把它理解成一种“切分后的词片段”。当论文说某个 token 概率很低时，意思就是当前模型很不习惯在这个位置说出这一步。

On-policy 的直观理解

模型今天生成了 8 个解法，它就从这 8 个解法的好坏里学。数据和当前策略高度绑定。

Off-policy 的直观理解

除了模型自己生成的数据，还允许把“别人生成的轨迹”拿来学，比如一个更强模型给出的成功解题过程。

一个关键误区

Off-policy 不等于简单地“拿别人的答案来监督训练”。这篇论文真正做的，是把别人的轨迹放进强化学习目标里，而不是只做前置的监督微调。

Stage 03

论文核心问题

作者究竟认为已有 RLVR 卡在哪里。

引言最重要的一句潜台词是：On-policy RL 往往只能放大模型已有的行为，而不是生成真正新的认知能力。换句话说，如果底模在某类推理模式上几乎完全不会，那你再让它自己反复采样和反馈，它也可能只是原地打转。

你可以把作者的论点理解成一句很朴素的话：学生自己做不出来的题，只靠自己不断试，很可能还是不会；这时候需要一个更强老师把正确思路显式摆在他面前。
原论文主张

RL 在这种设置下受限于 base LLM 本身，容易出现性能平台期，尤其当模型缺乏某些基础认知行为时更明显。

我来翻译

如果模型一开始就不具备某类推理模板，它可能连“踩中正确轨迹”的机会都很少，于是奖励信号几乎给不回来，训练自然推不动。

读到这里你该记住

LUFFY 不是在补一个工程小技巧，而是在挑战“只靠模型自己 roll out 就能学会推理”这个隐含前提。

Stage 04

方法总览

先看全貌，再进入 Sec 3 的细节公式。

LUFFY 由两个核心部件组成：Mixed-Policy GRPO 和 Policy Shaping via Regularized Importance Sampling。第一个负责把 on-policy 与 off-policy 轨迹真正混进同一个 RL 目标；第二个负责避免这种混合训练太快变成“机械抄老师”。

输入

题目、当前策略模型自己的 rollout、强模型提供的成功推理轨迹。

组内比较

把 on-policy 和 off-policy 放进同一组，用统一的奖励统计来算 advantage。

概率修正

用 importance sampling 修正不同数据来源带来的梯度偏差。

塑形

额外强调当前模型还不擅长、但在强轨迹中很关键的低概率动作。

这一节的核心直觉

如果模型自己做错，强模型轨迹会成为更优参考；如果模型自己也能做对，就保留它自己的探索空间。LUFFY 追求的不是单向模仿，而是动态切换重心。

Sec 2

背景精讲：RLVR 与 GRPO

这一节是方法部分的地基，不懂它就难看懂后面公式。

2.1 Verifiable Reward Function

Outcome Reward 指只根据最终结果给奖励，不直接对中间过程做人工打分。本文里的奖励最简单：最终答案对，奖励为 1；否则为 0。

论文里的奖励定义
R(τ) = 1, if τ outputs the correct final answer to q R(τ) = 0, otherwise

白话理解：这不是奖励“过程像不像人类”，而是奖励“最后答对没”。它简单、干净、可自动判断，所以很适合大规模训练。

2.2 GRPO 是什么

GRPO 全称是 Group Relative Policy Optimization。你可以把它看成一种“组内相对比较”的 RL 方法。对于同一道题，模型采样出一组解法，然后根据这组解法内部谁更好、谁更差来计算每条轨迹的 Advantage。

Advantage 可以先理解成“这条轨迹相对于同组其他轨迹，多值得被鼓励多少”。它不是绝对分数，而是相对位置。

GRPO 的 advantage 计算
A_i = (R(τ_i) - mean({R(τ_i)})) / std({R(τ_i)})

白话理解：如果一组样本里只有你答对了，而其他都错了，你的 advantage 就会比较高；如果大家都差不多，你的相对优势就不大。

为什么要用组内相对比较

这样可以避免额外训练一个 value model。论文这里沿用的是 RLVR 实践里已经比较常见的思路。

你真正该懂什么

GRPO 的关键不是名字，而是它把“同题多个解法的相对成败”变成了学习信号。

2.3 Importance Sampling 第一次出现

Importance Sampling 直译是“重要性采样”，这里更准确的理解是“分布校正”。因为当前在更新的策略 πθ 和生成这些轨迹时所用的旧策略 πθold 并不完全一样，所以梯度需要乘一个概率比值来修正。

在 GRPO 里出现的比值
r_{i,t}(θ) = π_θ(τ_{i,t} | q, τ_{i,<t}) / π_{θold}(τ_{i,t} | q, τ_{i,<t})

白话理解：如果当前策略比旧策略更喜欢这个动作，那这个比值更大；反之更小。它的作用是让更新更符合 policy gradient 的理论要求。

Sec 3

方法与公式拆解

这是整篇论文最关键的部分，也是你需要最耐心读的一节。

3.1 Mixed-Policy GRPO：先把两种轨迹混在一起

论文的第一步改动，是把 Off-policy Rollouts 直接并入用于组内比较的轨迹集合。这里的 Rollout 可以理解成“模型从题目出发一路生成到答案的完整轨迹”。

混合后的 advantage
Â_i = (R(τ_i) - mean(G_on ∪ G_off)) / std(G_on ∪ G_off)

白话理解：以前只跟自己生成的其他答案比较，现在还要跟强模型给出的答案一起比较。于是，当你自己都做错时，强模型轨迹会自然显得“更值得学”。

术语首次解释：Behavior Policy

Behavior Policy 指产生训练样本的那个策略。对 on-policy 样本来说，它接近旧策略；对 off-policy 样本来说，它来自另一个外部模型。

这一招真正妙在哪里

作者没有简单把强模型答案当“标准答案”硬塞给学生，而是让它们参与同一个相对比较机制，从而使模仿的强度随模型自身表现动态变化。

Mixed-Policy GRPO 总目标
J_Mixed(θ) = 1 / Z * ( Σ_off Σ_t CLIP(r̂_{j,t}(θ, ϕ), Â_j, ε) + Σ_on Σ_t CLIP(r_{i,t}(θ), Â_i, ε) )

白话理解：总目标由两部分组成，一部分是学习强模型轨迹，一部分是学习自己采样出来的轨迹。两种信号被放在同一个优化目标里。

3.1 里第二个关键点：off-policy importance sampling

对 off-policy 轨迹，论文引入了新的概率比值 r̂。这里的分母不再是旧策略，而是外部策略 πϕ。这一步反映的是：你现在在用“别人生成的数据”更新当前策略，所以需要校正两者的分布差异。

Off-policy 的概率修正
r̂_{j,t}(θ, ϕ) = π_θ(τ_{j,t} | q, τ_{j,<t}) / π_ϕ(τ_{j,t} | q, τ_{j,<t})

白话理解：如果当前模型对某个老师动作也越来越认可，这个动作的学习信号会更强；如果它离当前模型很远，更新就会经过校正，不会盲目照搬。

这里有一个工程上的简化

论文实践中直接采用 πϕ = 1，目的是方便整合不同来源的强模型轨迹，避免重新计算外部模型概率和处理不同 tokenizer 的复杂性。这是一个值得你以后继续追问的地方：它在工程上很实用，但和标准 off-policy 修正并不完全一样。

3.2 为什么还要 Policy Shaping

作者发现，只做 Mixed-Policy GRPO 会带来一个新问题：训练收敛更快了，但探索也更快消失了。这里的 Entropy 指策略分布的“发散程度”或“不确定性”。如果 entropy 很快掉下去，通常意味着模型越来越死板，只会选少数高概率动作。

作者观察到的现象

模型会优先强化那些“强模型轨迹里有、而且自己本来就不太陌生”的 token，却忽略那些真正关键但当前概率很低的 token。

我来翻译

学生最容易学到的，是老师答案里自己也差不多会写的部分；最难学到的，恰恰是自己最不熟的那几步关键思路。

3.2 Policy Shaping 的核心函数

所以作者引入一个 Shaping Function，把原本的 importance ratio 做一个变换，目的是给低概率但关键的动作更大的梯度权重。

Shaping 后的目标
J_Shaping(θ) = 1 / Z * ( Σ_off Σ_t f(r̂_{j,t}(θ, ϕ)) * Â_j + Σ_on Σ_t CLIP(r_{i,t}(θ), Â_i, ε) )

白话理解：对 off-policy 部分，不再直接用原始比值，而是用变换后的权重来重新分配学习力度。

作者选择的 shaping function
f(x) = x / (x + γ), γ = 0.1

白话理解：这个函数的目的不是简单放大所有 off-policy 信号，而是重新平衡梯度，让当前模型低概率区域也能得到更有力的学习。

你真正该怎么读 Eq. 7 和 Eq. 8

论文接着推导梯度，说明 shaping function 的导数 f'(πθ) 实际上在决定不同动作的梯度权重。你不必一开始就完全推懂推导细节，但要抓住作者想说明的逻辑：

  1. 如果直接用线性函数 f(π)=π，当动作概率很低时，梯度容易太小。
  2. 而论文希望的恰恰是：对那些当前模型还很不会、但强轨迹中很重要的动作，给予更多学习力度。
  3. 于是 shaping 本质上是在重新分配“该把注意力放在哪些 off-policy 动作上”。
Sec 3 的一句话总结

Mixed-Policy GRPO 解决“怎么把强模型轨迹真正混入 RL”；Policy Shaping 解决“混进来以后别学成死板模仿”。

Formula Lab

逐公式精读

把关键公式拆成“变量是什么意思、这项为什么存在、删掉会怎样”。

Eq. 2：组内 Advantage
A_i = (R(τ_i) - mean({R(τ_i)})) / std({R(τ_i)})

这一步把“绝对对错”转成“相对值得鼓励多少”。如果去掉均值和标准差，模型就只知道对错，不知道自己在同组里赢了多少。

Eq. 5：Mixed Objective
J_Mixed(θ) = on-policy term + off-policy term

你可以把它理解成“两个老师一起给分”：一个是当前模型自己的试错轨迹，一个是外部强模型的成功轨迹。

Eq. 6：Shaping Objective
J_Shaping(θ) = f(r̂) * Â + on-policy term

这一式子的关键不是形式，而是策略：把 off-policy 学习重点转向“模型当前还不熟”的那部分。

Eq. 2 应该怎么看

变量表

τ_i 是第 i 条轨迹，R(τ_i) 是它的最终奖励，mean 和 std 是这一组轨迹奖励的均值和标准差。

真正作用

它把“对不对”变成“在这一题里你比同伴强多少”。这就是为什么 GRPO 的学习信号本质上是组内相对信号，而不是全局绝对信号。

Eq. 5 应该怎么看

这条式子最应该盯住的是“off-policy 项终于进入了正式 RL 目标函数”。如果没有它，强模型轨迹通常只能以 SFT 数据的形式出现；而有了它，强轨迹不再是训练前的一次性热身，而是训练过程中的持续指导。

如果去掉 off-policy 项会怎样

模型就会退回到纯 on-policy RLVR，只能依靠自己已经能采样到的轨迹学习。对弱模型来说，这常常意味着根本拿不到足够的正反馈。

Eq. 6 到 Eq. 8 应该怎么看

这几条式子连起来看的重点是：作者不是说“多学一点老师答案”就够了，而是说“要把梯度预算优先花在最值得补的低概率动作上”。这是一种非常具体的训练偏置设计。

推荐记忆法

Eq. 2 负责“谁更值得鼓励”；Eq. 5 负责“把谁放进训练目标”；Eq. 6 到 Eq. 8 负责“该把更大的学习力气花在哪些动作上”。

Sec 4

实验设置

实验部分不是陪衬，它决定了你该不该相信前面的论点。

训练数据

作者使用 OpenR1-Math-220k 的一个子集，原始默认子集有约 94k prompts，过滤掉过长和验证错误的强模型轨迹后，最终保留约 45k 条训练样本。这里的 off-policy reasoning traces 来自 DeepSeek-R1。

这意味着什么

LUFFY 的成功很大程度上依赖高质量强模型轨迹，所以你不能把它误解成“单靠训练框架就自然学会了”。数据质量本身是这个方法成立的重要前提。

训练配比

每题总共用 8 个 rollout。纯 on-policy 基线是 8 条 on-policy；LUFFY 则使用 1 条 off-policy 加 7 条 on-policy。这一点很关键，因为它说明作者不是把训练变成“老师答案主导”，而是只插入少量外部指导。

评测任务

In-distribution 数学基准包括 AIME 2024/2025、AMC、Minerva、OlympiadBench、MATH-500。OOD 评测包括 ARC-c、GPQA-diamond 和 MMLU-Pro。

为什么要看 In-distribution

看方法在目标任务域里到底涨了多少。

为什么要看 OOD

看模型到底学到了可迁移的推理能力，还是只学会了训练数据风格。

Sec 5

结果解读

结果不只要看高低，还要看它们支持了哪条论证链。

5.1 主结果：平均提升明显

在 Qwen2.5-Math-7B 上，LUFFY 在 6 个数学 benchmark 上的平均分达到 50.1，比之前 RLVR 方法平均高约 6.4 分，也比作者自己复现的 on-policy RL 高约 4.6 分。单看数字，它已经很强；但更重要的是，它在 AIME 2025 这种更具挑战性的测试上展现了明显优势。

你应该怎么解读这组结果

作者想证明的不是“off-policy 也能用”，而是“off-policy guidance 真的能帮助模型学到一些单靠自己 roll out 不容易学到的模式”。分数提升是这个主张的外在证据。

5.1 OOD 结果：这是非常关键的一点

LUFFY 在 ARC-c、GPQA、MMLU-Pro 上的平均 OOD 成绩也优于最佳对比 RLVR 方法约 6.2 分。这一点非常重要，因为如果它只是死记强模型风格，通常不太容易在分布外任务上表现这么稳。

但也不能过度解读

OOD 提升不自动等于“真正学会了通用推理”。它只是说明这个训练策略比简单 SFT 或纯 on-policy 更有机会获得迁移收益。

5.2 弱模型实验：整篇论文最有分量的证据之一

作者特别强调，在 LLaMA-3.1-8B 上，纯 on-policy RL 在 Hard 训练集上奖励会掉到接近 0，也就是几乎学不动；而 LUFFY 在 Easy 和 Hard 上都能稳定提升。这直接支撑了论文的中心论点：当模型自己摸不到正反馈时，外部指导能够提供必要的支架。

5.3 训练动态：为什么作者坚持“不是盲目模仿”

论文展示的训练动态表明，LUFFY 早期会更像在模仿强模型轨迹，生成长度逐渐接近 off-policy reasoning traces；但随着训练进行，模型自己的 on-policy rollout 逐渐变得更有价值，于是探索重新变重要。

这说明了什么

作者想强调的是一种“先借外力起步，再逐步把外部模式内化为自己的能力”的学习过程，而不是全程照抄老师答案。

Index

术语索引

把这篇文章里最常见的概念压成一张可回看的词表。

RLVR

带可验证奖励的强化学习。奖励通常只看最终答案是否正确。

On-policy

训练数据主要来自当前模型自己生成的轨迹。

Off-policy

训练允许使用别的策略或别的模型产生的数据。

GRPO

按同题多个解法的相对成败来计算 advantage 的 RL 方法。

Advantage

某条轨迹相对于同组其他轨迹，多值得被鼓励多少。

Importance Sampling

当训练数据和当前策略不完全同分布时，用概率比值做分布校正。

Behavior Policy

产生训练样本的策略。对 off-policy 轨迹来说，它不是当前训练策略。

Entropy

策略分布的不确定性或发散程度。掉得太快常意味着探索塌缩。

Policy Shaping

通过变换 importance ratio，重新分配 off-policy 学习时的梯度权重。

怎么使用这张词表

如果你哪一段读到卡住，不要硬往下推。先回来查它属于“奖励定义”“组内比较”“分布校正”还是“探索保持”中的哪一类，再回去看原段落，理解会顺很多。

这篇论文的价值与局限

真正的科研阅读，不是只接受结论，还要学会追问边界。

我认为它最有价值的地方
  1. 它明确提出了纯 on-policy RL 的能力上限问题。
  2. 它把强模型轨迹整合进 RL，而不是只停留在 SFT 蒸馏。

我认为你该继续追问的地方
  1. 效果有多少来自 LUFFY 框架本身，有多少来自强轨迹质量？
  2. 只看最终答案对错，是否足以说明中间推理真的更好？
  3. 数学任务上的成功，能否稳定迁移到代码、科学推理、规划等场景？
